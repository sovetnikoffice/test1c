Таблица внедрения подсистем позволяет зафиксировать принятые решения по внедрению в удобном матричном виде. В колонке слева перечислены объекты конфигурации, к которым возможно подключить те или иные функции подсистем, а сверху - список этих функций. В ячейках таблицы с помощью двойного щелчка мыши устанавливается или снимается отметка о принятом решении. Для каждой ячейки таблицы имеется возможность комментировать и обсуждать принятые решения с помощью примечаний. С помощью отбора по объектам метаданных и по внедряемым подсистемам можно уменьшить область до интересующего фрагмента таблицы внедрения. При завершении работы с таблицей, принятые решения будут сохранены, а также их можно экспортировать во внешний mxl-файл или распечатать.

Кроме того, в этой же форме можно сверить текущее состояние конфигурации с принятыми проектными решениями. Для этого после простановки всех отметок соответственно принятым проектным решениям, необходимо выполнить команду "Проверить внедрение".

Ячейки таблицы будут раскрашены в соответствии с результатами проверки внедрения подсистем БСП, исходя из отмеченных проектных решений.

Легенда таблицы внедрения:

+ Принято решение по внедрению
(было ранее зафиксировано или зачитано из конфигурации).
Решение по внедрению не принято или принято отрицательное решение 
(было ранее зафиксировано или зачитано из конфигурации).
+ Ошибок при внедрении не обнаружено. Однако это не означает, что внедрение выполнено корректно. 
Могут остаться места,  которые возможно проверить только вручную.
+ Проектное решение реализовано некорректно. 
Например, внедрение подсистемы произведено не полностью.
Проектное решение не принято, 
однако в конфигурации была ошибочно выполнена какая-то часть работы по внедрению.
Проектное решение не принято (или принято отрицательное решение), 
однако в конфигурации имеется корректная реализация.

 

Существует возможность быстрого отбора только по ошибочным случаям внедрения подсистем БСП в объекты конфигурации. Чтобы ей воспользоваться, необходимо в поле "Отбор по объектам" установить значение "Только ошибки".

После проверки внедрения в каждом ошибочном случае есть возможность посмотреть описание ошибки с точностью до реквизита (свойства). Для этого необходимо вызвать у ячейки с требуемой ошибкой команду контекстного меню "Показать описание ошибки".
